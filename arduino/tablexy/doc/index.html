Documentation

grblServer.py is the interface with the motors. It receives commands and executes them. To launch it use 

python3 grblServer.py -u /dev/ttyACM0
Other options are shown with 
python3 grblServer.py -v
Connect to the server using 
python3 client.py

Debugging

The grbl 0.8 documentation is here. To connect directly with the grbl card, use screen. The usage is the following: 
screen -L /dev/ttyACM0 9600
Typing in the window does not echo. Type 
$X
to unlock the card. 
$$
shows the current configuration. grbl commands can be typed in the screen and are soon executed. Once finished, type 
Ctrl+A, Ctrl+D
Then kill the connection, if needed, 
cmsdaq@pccmsdaq01:~/TableXY$ screen -ls
There is a screen on:
	26134.pts-24.pccmsdaq01	(25/01/2019 09:46:54)	(Detached)
1 Socket in /var/run/screen/S-cmsdaq.

cmsdaq@pccmsdaq01:~/TableXY$ screen -X -S 26134.pts-24.pccmsdaq01 quit
cmsdaq@pccmsdaq01:~/TableXY$ screen -ls
No Sockets found in /var/run/screen/S-cmsdaq.
